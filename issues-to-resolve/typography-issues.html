Typography issues

Reported issues include:

  • Too much space between lines, not enough between paragraphs
  • List spacing is unpredictable
  • Default button text is smaller than body
  • Special pages like topics have particular sizing that is hard to get applied consistently
  • Sidebar H2 might be too large

Standard paragraphs

The line spacing is slightly larger than typical, but there is evidence to support that it's easier to read this way.

The problem comes from the spacing between paragraphs not being sufficiently differentiated. If you can't tell is something is a new paragraph or not then it's likely to actually hurt readability. Now I'm just adding more text to so it stretches to an extra line in the paragraph. It's more realistic to have some longer paragraphs here and there, isn't it?

To that end, we need to increase the space between typical paragraphs.

Current spacing

Right now, in Mac Chrome at standard sizes, we have:

  • Line height: 22 pixels between the median of one line and the baseline of the line above. This is specified as 1.75em, which should equate to 175% of the font size
  • Paragraph spacing: 28 pixels between the median of one line and the baseline of the paragraph above.
  • That makes paragraph spacing 127% of line spacing.

Spacing on similarly large line-spacing sites

Medium has 22 px between lines, and 52 px between paragraphs - 236%

A List Apart has 21 px between lines, and 45 pixels between paragraphs - 214%

Smashing magazine has 26 px between lines, 57 between paragraphs - 219%

Safari reading mode gives you 20 px between lines, and 55 px between paragaphs - 275%

Adjustment needed

If we wanted say, 220% of the line height betwen paragraphs, we'd need it to be: 48 px instead of the current 33 px.

With 1em of margin-bottom on the p, we get 42 pixels.
