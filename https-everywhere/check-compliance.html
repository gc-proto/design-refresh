Compliance definitions

Table columns report compliance with a section of the policy direction.

Each column shows an overall compliance percentage for each organization or domain. Click “show details” or “show subdomains” to view the column values for individual domains.

ITPIN Compliant

Whether a domain is compliant with the entire ITPIN 2018-01 direction.

Values

  • Yes: the domain is compliant with ITPIN 2018-01.
  • No: the domain is not compliant with ITPIN 2018-01.

Enforces HTTPS

Whether a domain uses and enforces HTTPS by default. This can be done by redirecting HTTP endpoints to HTTPS, or by only being available over HTTPS.

Values

  • Yes: the domain does enforce HTTPS by default.
  • No: the domain does not enforce HTTPS by default.

HSTS

Whether a domain has implemented HTTP Strict Transport Security, which ensures that modern web browsers will only communicate with the domain over HTTPS (even if the user tries to access an HTTP link).

Values

  • Yes: a valid Strict-Transport-Security header with a max-age value of at least 1 year (listed as 31536000 seconds) is present on the domain’s default endpoint.
  • No: the Strict-Transport-Security header is either missing our using a max-age of less than 1 year.
  • Preloaded: the subdomain is considered to be HSTS-compliant due to its parent domain having been preloaded in modern browsers.

Free of known weak protocols and ciphers

Whether a domain is free of known weak protocols and ciphers, including RC4 and 3DES ciphers and SSLv2, SSLv3, TLSv1.0 and TLSv1.1 protocols.

Values

  • Yes: the domain’s primary endpoint does not use the RC4 or 3DES ciphers, and has disabled the SSLv2, SSLv3, TLSv1.0 and TLSv1.1 protocols.
  • No, download CSV for details: known-weak ciphers and/or protocols are still in use, and the CSV download will list the deprecated ciphers and/or protocols.
  • N/A: protocols and ciphers are not applicable for this domain since it does not offer HTTPS

Uses approved certificates

Whether a domain uses approved certificates as outlined in the Communication Security Establishment’s Guidance on Securely Configuring Network Protocols (ITSP.40.062).

Values

  • Yes: The domain uses an approved certificate
  • No, download CSV for details: unapproved certificates are still in use, and the CSV download will list the unapproved certificate.
  • N/A: certificates are not applicable for this domain since it does not offer HTTPS
